#5052 전화번호 목록 (G4)
    - 문자열 문제
    - unordered_set 자료구조를 사용해서 문제를 해결하면 됨
    - unordered_set : 정렬하지 않은 set
        - 해쉬테이블 형태로 노드를 저장함
        - 그렇기 때문에, 삽입, 검색 등의 시간복잡도가 O(1)임
    - set : 정렬되어 있는 set
        - 레드블랙트리 형태로 노드를 저장함
        = 그렇기 때문에, 삽입, 검색 등의 시간복잡도가 O(logN)임
    --> 따라서 항상 정렬된 상태를 유지해야하는 경우 : set, 그렇지 않은 경우 : unordered_set을 사용하면 됨
